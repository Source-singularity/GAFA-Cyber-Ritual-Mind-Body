这是一份为您量身定制的**《多维融合电机运动控制微调指南》**。

这套系统的核心逻辑基于一个**乘法模型**：
> **实际运行一个周期的时间 = 基准时间(6秒) ÷ ( 心率因子 × 情绪因子 )**
> *(注：时间越短，电机转速越快，运动越剧烈)*

所有的修改都在 STM32 的 `main.cpp` 文件中的 `calculateMotionParameters()` 和全局变量区域。以下是具体参数的调整方法和对应效果：

---

### 一、 基础参数微调 (决定动作幅度和基准手感)

**1. 修改运动行程（距离）**
*   **代码位置**：全局变量区 `const long STEPS_PER_TRIP = 10000;`
*   **怎么调**：
    *   如果觉得 10cm 太长，想改成 **5cm**：改为 `5000`。
    *   如果想拉满 T8 丝杆行程到 **15cm**：改为 `15000`。
    *   *注意：改动距离后，电机走完单程的速度会发生视觉变化（因为要在相同时间内走完不同距离）。*

**2. 修改基准周期时间（默认速度）**
*   **代码位置**：`calculateMotionParameters()` 中的 `currentCycleTime = 6.0 / (hr_factor * emo_factor);`
*   **怎么调**：
    *   目前的基准是 **6.0秒** 跑完一上一下。
    *   如果觉得平时整体太慢，把 `6.0` 改成 `4.5` 或 `5.0`。
    *   如果觉得平时太快有噪音，改成 `8.0`。

---

### 二、 心率因子 (HR Factor) 微调指南

心率控制使用的是**线性映射 (`mapFloat`)**，这意味着速度是丝滑过渡的，而不是突变的。

**代码位置**：
```cpp
  if (currentBPM >= 40 && currentBPM <= 60) hr_factor = 2.0;
  else if (currentBPM > 60 && currentBPM < 80) hr_factor = mapFloat(currentBPM, 60, 80, 2.0, 1.0);
  else if (currentBPM >= 80 && currentBPM <= 90) hr_factor = 1.0;
  else if (currentBPM > 90 && currentBPM <= 130) hr_factor = mapFloat(currentBPM, 90, 130, 1.0, 0.25);
```

**怎么调**：
*   **想让低心率（60以下）时电机更疯狂**：把第一行的 `2.0` 改成 `2.5` 甚至 `3.0`。同时记得把第二行 `mapFloat` 里的 `2.0` 也同步改掉，保持过渡平滑。
*   **想扩大“不加速也不减速”的平稳区**：目前平稳区是 80~90。如果你希望 75~100 都不变速，改成：
    `else if (currentBPM >= 75 && currentBPM <= 100) hr_factor = 1.0;`
*   **想让高心率（130）时电机几乎停滞**：把最后一行 `mapFloat` 的 `0.25` 改成 `0.1`（此时若无情绪干预，周期会长达 6/0.1 = 60秒，变成极其缓慢的龟速蠕动）。

---

### 三、 情绪因子 (Emo Factor) 与 周期数 微调指南

情绪是**突发干预**，它会瞬间成倍放大或缩小当前的心率速度，并锁定执行几次。

**代码位置**：
```cpp
  if (currentEmotion == "anger" || currentEmotion == "fear") { emo_factor = 2.0; cycles = 2; }
  else if (currentEmotion == "contempt" || currentEmotion == "disgust") { emo_factor = 1.5; cycles = 1; }
  else if (currentEmotion == "neutral") { emo_factor = 1.0; cycles = 1; }
  else if (currentEmotion == "happiness" || currentEmotion == "sadness") { emo_factor = 0.7; cycles = 1; }
  else if (currentEmotion == "surprise" || currentEmotion == "surpris") { emo_factor = 0.5; cycles = 2; }
```

**怎么调**：
*   **调整爆发力 (`emo_factor`)**：
    *   如果希望 `anger` (愤怒) 时具有恐怖的爆发力，把 `2.0` 改成 `3.0`。
    *   如果希望 `sadness` (悲伤) 时机器显得极度无力，把 `0.7` 改成 `0.3`。
*   **调整情绪锁定时间 (`cycles`)**：
    *   目前 `anger` 触发后，机器会以高转速连续暴走 **2个周期** (上-下-上-下) 才重新听从指令。
    *   如果你希望它暴走得更久（比如让人产生压迫感），把 `cycles = 2` 改为 `cycles = 3` 或 `4`。
    *   如果你希望它只抽搐一下就立刻恢复，把所有的 `cycles` 都改成 `1`。

---

### 四、 机械安全限制 (非常重要！)

当你把心率因子和情绪因子都调得很大时（比如 HR是2倍，情绪是2.5倍，叠加就是5倍速，原6秒周期会被压缩到1.2秒），**步进电机会因为加速度过大而发出“叽叽叽”的惨叫声，并且卡在原地不动（这叫丢步/堵转）**。

因此，必须设置安全底线。

**1. 物理极限防堵转限速**
*   **代码位置**：`if (currentCycleTime < 2.5) { currentCycleTime = 2.5; }`
*   **怎么调**：
    *   **向上试探**：如果你发现偶尔电机还是会尖叫卡死，说明你的机械结构阻力较大，承受不了 2.5秒跑完一圈。把 `2.5` 改成 `3.0` 或 `3.5`。
    *   **向下压榨**：如果你的丝杆抹了润滑油，极其顺滑，你想看它能跑多快。你可以把 `2.5` 改成 `1.8` 或者 `1.5`，然后故意触发 `anger` 情绪去测试极限。一旦出现卡死声，立马调高 0.3。

**2. 极端生理安全模式**
*   **代码位置**：`if (currentBPM > 130 || currentBPM < 40) { currentCycleTime = 15.0; targetCycles = 1; return; }`
*   **怎么调**：
    *   这段代码是凌驾于所有逻辑之上的。如果检测到极度异常的心率，机器会进入“安抚模式”。
    *   如果不想这么慢，把 `15.0` 改成 `10.0`。
    *   如果你想在极端心率下让机器**完全停止**，可以直接加一句 `currentCycleTime = 9999.0;`，或者干脆让电机禁用。

### 五、 终极避坑建议（关于硬件微调）

如果在软件怎么调都觉得不舒服，请调整硬件：
1.  **觉得运动有“颗粒感/震动感”？**
    *   把 TB6600 的细分从 **800** 拨到 **1600** 或 **3200**。
    *   **必须同步修改代码**：`STEPS_PER_TRIP` 也要成倍增加。比如细分变 1600，`STEPS_PER_TRIP` 就得改成 `20000`。这样运动会变得极其安静丝滑！
2.  **觉得电机力气不够，总卡住？**
    *   在 TB6600 上把电流从 1.0A 微微调大到 1.5A 或 1.7A（注意前文提到的温度问题，烫手正常，别化了支架就行）。
