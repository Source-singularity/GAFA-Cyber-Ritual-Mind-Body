# 🧠 (GAFA意智与异质)生理-情绪融合交互系统 (AI Lie Detector & Bio-Reactor)

基于 **视觉面部情绪识别 (HSEmotion AI)** 与 **指尖心率血氧监测 (MAX30105)** 的双重生物反馈系统。系统会根据被测者的**真实心率**与**潜意识微表情**，实时控制步进电机与丝杆机构的运动频率与幅度，打造极具压迫感与科技感的“赛博朋克测谎仪”物理交互体验。

## ✨ 核心功能
*   👀 **AI 情绪捕捉**：调用 PC 端 GPU (如 RTX 5080) 实时运行 HSEmotion 模型，精准识别 8 种面部情绪。
*   🫀 **物理心率监听**：STM32 边缘实时解算 MAX30105 脉搏波形，提取真实 BPM。
*   ⚙️ **动态机械反馈**：42步进电机结合 T8 丝杆，心率越低越疯狂，情绪越激烈动作越暴躁。
*   🛡️ **防死锁状态机**：后台硬件定时器发脉冲，电机运动与心率采样绝对隔离，互不干扰。

## 🔌 2. 硬件接线指南（小白必看避坑指南）

⚠️ **警告：接线时请务必断开所有电源！特别是电机的 12V/24V 电源，带电插拔电机线会导致 TB6600 瞬间烧毁！**

### 模块 A：心率传感器 (MAX30105) -> STM32
采用 I2C 通讯，线材尽量短。
*   `VIN`  ->  `3.3V` 或 `5V` (根据你的模块版本)
*   `GND`  ->  `GND`
*   `SCL`  ->  `PB6` (STM32 默认 I2C1_SCL)
*   `SDA`  ->  `PB7` (STM32 默认 I2C1_SDA)

### 模块 B：步进电机驱动 (TB6600) -> STM32  *(采用共阳极接法)*
此接法最稳定，能利用 5V 提供充足的光耦驱动电流。
1.  **控制信号端 (接单片机)：**
    *   将 TB6600 上的 `PUL+`、`DIR+`、`EN+` 用线**全部短接在一起**，然后连到 STM32 开发板的 **`5V`** 引脚。
    *   `PUL-` (脉冲) ->  **`PB12`**
    *   `DIR-` (方向) ->  **`PB13`**
    *   `EN-`  (使能) ->  **`PB14`**
2.  **电机端 (接42步进电机)：**
    *   `A+` 和 `A-` 接电机的一相（例如 黑、绿）
    *   `B+` 和 `B-` 接电机的另一相（例如 红、蓝）
    *   *小白找相技巧：随便拿两根电机线碰在一起，如果拧转电机轴觉得阻力突然变大，这两根就是同一相！*
3.  **电源端：**
    *   `VCC` -> 接 12V 或 24V 直流电源正极
    *   `GND` -> 接 直流电源负极

### 模块 C：TB6600 侧面拨码开关设置
*   **细分 (Microstep)**：拨到 **`800`** 这一档 (通常是 S1, S2, S3 对应的开关组合，请看驱动器外壳表格)。
*   **电流 (Current)**：拨到 **`1.5A`** (或最接近的 1.2A/1.7A 档位)。

## 💻 3. 软件环境搭建与运行

### 第一步：烧录 STM32 下位机 (PlatformIO)
1. 使用 VSCode 打开 `HeartRate_Device` 文件夹。
2. 确保已安装 **PlatformIO** 插件。
3. 在 `platformio.ini` 的 `lib_deps` 中搜索并安装 `SparkFun MAX3010x Pulse and Proximity Sensor Library`。
4. 将 STM32 通过串口模块 (如 CH340) 连接电脑，点击底部的 `Upload` 箭头烧录程序。

### 第二步：配置 Python 上位机环境
1. 打开终端，进入 `LieDetector_AI` 文件夹。
2. 安装依赖库：
   ```bash
   pip install torch torchvision torchaudio opencv-python pyserial numpy
   ```
3. 确保你的 HSEmotion 权重文件 `enet_b2_8.pt` 放在了代码指定的目录 (`lie_detector_env/.hsemotion/`) 下。

### 第三步：让系统跑起来 🚀
1. 给步进电机的 TB6600 驱动器通上 12V/24V 电源。
2. 将 STM32 的 USB 串口插上电脑（此时它应该被识别为 `/dev/ttyUSB0` 等）。
   *(Linux 用户如果遇到权限报错，请执行 `sudo chmod 666 /dev/ttyUSB0`)*
3. 运行 Python 脚本：
   ```bash
   python final_stable_lie_detector.py
   ```
4. **互动开始**：将手指轻轻按在 MAX30105 传感器红灯上，注视电脑摄像头。当屏幕上出现心率波形后，电机将开始随你的心跳和表情起舞！

## 🛠️ 4. 核心逻辑与参数微调指南 (Tuning Guide)

想要机器更狂暴？或是更温和？你只需要修改 STM32 `main.cpp` 中的几行代码。

机器单次往复时间公式：
> **实际运动时间 = 基准时间(6秒) ÷ ( 心跳倍率 × 情绪倍率 )**
> *(注：时间越短，电机转速越快，看起来越疯狂)*

打开 `main.cpp`，找到 `calculateMotionParameters()` 函数：

### 🔧 调整 1：修改运动行程 (动作幅度)
找到最上方的全局变量：
```cpp
const long STEPS_PER_TRIP = 10000; // 默认单程 10cm (12.5圈 * 800细分)
```
*   想改成 **5cm** 的短促活塞运动？改为 `5000`。
*   想改成 **15cm** 的大开大合？改为 `15000`。

### 🔧 调整 2：修改情绪爆发力
找到代码中的情绪判断区：
```cpp
if (currentEmotion == "anger") { emo_factor = 2.0; cycles = 2; }
```
*   **`emo_factor` (爆发速度)**：`2.0` 代表 2 倍速。如果你希望它识别到愤怒时变得极其狂暴，改成 `3.0`。
*   **`cycles` (情绪锁定时长)**：`2` 代表它会以这个狂暴速度连续抽插 2 次才停下。觉得不够压迫？改成 `4`。

### 🔧 调整 3：解开硬件速度封印 (⚠️ 危险)
机器设置了物理安全底线，防止电机烧毁或卡死：
```cpp
if (currentCycleTime < 2.5) { currentCycleTime = 2.5; }
```
*   这表示它**最快只能 2.5 秒**跑完一上一下。
*   如果你的丝杆抹了大量润滑油，且觉得还不够刺激，可以尝试把 `2.5` 降低到 `1.8`。
*   *后果：如果电机发出“叽叽叽”的惨叫且原地不动，说明超出了物理极限（丢步了），请乖乖改回 2.5。*

## ❓ 5. 常见问题排查 (FAQ)

**Q1：电机表面非常烫，摸着烫手，是不是要烧了？**
> **完全正常！** 步进电机的特性决定了它即使停在原地也会通满电流锁死。只要滴水上去没有立刻发出“嘶嘶”声沸腾（不超过100℃），电机就很健康。
> *⚠️ 注意：如果你用 3D 打印的 PLA 材料做电机支架，60℃ 就会让 PLA 变软变形，建议垫一块隔热垫，或用 PETG 打印支架。*

**Q2：PC 屏幕上没有心率波形，一直是一条绿线？**
> 1. 检查手指是否放偏了，指腹要严实地盖住传感器的发光口，但**不要用力死死按住**（会把毛细血管血液挤走）。
> 2. 检查 I2C 的四根线是否松动。

**Q3：识别到愤怒时，电机发出刺耳的尖叫声且卡在原地不转？**
> 典型的**步进电机堵转/失步**现象。
> 原因：加速度太大，电机拖不动丝杆了。
> 解决：检查机械结构是否卡涩，或者在代码里把 `2.5` 秒的安全限速提高到 `3.0` 秒。

**Q4：我想让机器运转得更静音、更丝滑怎么做？**
> 将 TB6600 驱动器的拨码开关细分从 `800` 调高到 `1600`。
> **注意：** 必须同步在代码中将 `STEPS_PER_TRIP` 从 10000 改为 20000，否则单程距离会缩水一半！
