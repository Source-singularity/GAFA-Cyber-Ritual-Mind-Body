## ⚙️ 系统架构与核心技术 (System Architecture)

全面升级至 ESP32-S3 双核处理器，实现**零延迟无线局域网闭环控制**与**工业级单电源架构**。

1. **零束缚网络层 (Wi-Fi AP + UDP)**：
   * 祭坛（ESP32）自建极速局域网热点 `Cyber-Ritual`。（默认密码 12345678）
   * 摒弃传统的 USB 串口与 TCP 握手，采用 `UDP` 协议在空气中以毫秒级延迟双向倾泻心跳波形与情绪指令。
2. **多模态输入与决策层**：
   * **PC 端**：无线接收脉搏波，并调用 GPU 运行 HSEmotion 模型推断 8 种面部情绪，将结果 UDP 回传。
   * **ESP32 端**：自带 FPU 硬件浮点加速心率解算。内置状态机根据接收到的情绪覆盖本地速度因子。
3. **物理输出层 (IRAM 驻留级中断)**：
   * **穿刺模拟 (步进电机)**：使用 ESP32 专属高精度定时器（`timerAlarmWrite`），ISR 中断函数驻留内存（`IRAM_ATTR`），确保 10000 个高频脉冲在并发 Wi-Fi 通信时绝不撕裂卡顿。
   * **抽动模拟 (电磁铁)**：精准捕捉每一次脉搏波峰（`diff < 0`），驱动 12V 电磁铁产生 100ms 撞击，模拟伴随心跳的抽动痉挛。

* **电源核心**：12V 5A 电源适配器 + **XL4015 5A 降压模块 (带数显)**

## 🔌 工业级单电源布线指南 (Wiring Guide)

⚠️ **极度危险：请严格遵守以下降压模块调校顺序，否则 ESP32 将瞬间烧毁！**

### 步骤一：调校 XL4015 降压模块 (防烧毁必做)
1. **空载通电**：只将 12V 电源接入 XL4015 的 `IN+` 和 `IN-`。**输出端绝对不要接 ESP32！**
2. **设定恒压 (CV)**：用十字螺丝刀旋转靠近输出端的电位器，直到数码管显示精确的 **`5.0V`**。
3. **解禁恒流 (CC)**：将另一个调电流的电位器**顺时针狂拧 15 圈到底**。必须彻底解除限流保护，否则 ESP32 开启 Wi-Fi 瞬间会因电流受限导致电压跌落、无限重启。

### 步骤二：全局拓扑接线
所有供电均由单一 12V 墙插提供，清爽优雅。
```text
墙插 220V ──▶ [ 12V 5A 电源适配器 ]
                ├── 12V 线路 ──────────────▶ TB6600 (VCC/GND 供电)
                ├── 12V 线路 ──────────────▶ MOS 模块 (VIN+/VIN- 供电)
                │
                └── 12V 线路 ──▶ [ XL4015 降压模块 IN端 ]
                                        │
                                 (已锁死 5.0V)
                                        │
                                        └── USB / OUT+ ──▶ ESP32-S3 (供电)
                                                                 └── 3.3V 线条 ────▶ TB6600 (PUL+, DIR+, EN+ 共阳极供电)
```

### 步骤三：信号控制线对照表 (ESP32-S3)
| 模块端口 | ESP32-S3 引脚 | 备注 |
| :--- | :--- | :--- |
| **MAX30102 SCL** | **IO 9** | I2C 时钟 |
| **MAX30102 SDA** | **IO 8** | I2C 数据 |
| **MOS 模块 SIG(PWM)**| **IO 4** | 控制电磁铁抽动 |
| **TB6600 PUL-** | **IO 5** | 步进电机脉冲 |
| **TB6600 DIR-** | **IO 6** | 步进电机方向 |
| **TB6600 EN-** | **IO 7** | 步进电机使能 (可选) |

## 🛠️ 环境部署与运行 (How to Run)

1. **烧录 ESP32**：
   * 使用 VSCode + PlatformIO。
   * **关键**：确保 `platformio.ini` 中已添加 `-DBOARD_HAS_PSRAM` 与 `board_build.arduino.memory_type = qio_opi` 以激活 N16R8 的高配内存。
   * 点击 Upload 烧录。
2. **祭坛就位**：拔掉所有 USB 线，仅接通 12V 电源，XL4015 点亮，ESP32 启动热点。
3. **建立连接**：笔记本电脑连接名为 `Cyber-Ritual` 的 Wi-Fi。
4. **启动 AI**：
   ```bash
   python final_stable_lie_detector.py
   ```
5. **开始仪式**：手指触碰传感器。波形将凌空飞向屏幕，机器将彻底接管你的生理节律。

## 🎛️ 终极微调指南 (Tuning & Calibration)

### 1. 穿刺控制：深渊模式与狂暴模式
* **深渊窒息模式 (待机与惩罚)**：无人触碰或极端心率 (`>130`/`<40`) 时，系统强制进入 `MAX_CYCLE_TIME = 30.0`。机器将以极度压抑的速度进行长达 30 秒的深渊级蠕动，带来巨大的心理压迫感。
* **情绪暴走倍率**：在 `calculateMotionParameters()` 中，可自由定义情绪因子。如 `anger` 触发 3 倍速连续 4 个周期的狂暴抽插。
* **物理极限防堵转**：`MIN_CYCLE_TIME = 2.5` 设定了最高爆发速度，防止电机尖叫失步。

### 2. 电磁铁：力度与痉挛感
* **通电时长**：`SOLENOID_PULSE_MS = 100`。可通过调低至 60ms 获得更干脆的击打声。切勿超过 200ms 以免线圈积热烧毁。
* **机械调校**：用物理垫片缩短电磁铁铁芯的退回间隙至 5mm，爆发力将呈指数级增强。

### 3. 硬核物理能耗监控
* 按下 XL4015 降压模块的左侧按钮切换至 **“A (电流)” 显示模式**。
* 展览期间，观众将实时看到随心跳与 Wi-Fi 吞吐而剧烈跳动的物理能耗数字（如 `0.23A` → `0.58A`），为装置增添极强的工业与生命体征隐喻。
